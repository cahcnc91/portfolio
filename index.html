PuppyDate

This a fully responsive app that mimics Tinder usage, but for puppies, built using Nodejs, React, Redux and MongoDB.
Just like Tinder, the user can go through profiles and choose to match or not with another puppy. In case of a match for both, a chat is created and the users can talk to set up playdates for the puppies.
Real-time chating feature is also available, using Socket.io.
All styling using Bootstrap.

Think Empire

This is a data-driven app that combines the use of Google Maps and NYC real estate data.

I have been working full time for this company for the past year as a Full Stack Developer, creating and updating current UI, APIs, documentation and tests for a range of different features.

Problem: Time of load of all information for a property was too long.

Solution: Refactored state in redux store to reduce the amount of re-renders of components, class based components into to functional, and loading pattern upon request (when clicked).

Problem: The user needs the hability of filtering properties based on different criteria.

Solution: Every criteria selected gives a real-time count of the amount of results that will be displayed. After the user hits filter, the UI shows both a list and correspondent pins on map for properties available in the database that fit those criteria, there is pagination and load balancer, so only 20 properties are loaded at a time.

Problem: Hability to save properties to lists and create saved seaches that save criteria of searches.

Solution: Create, share, retrive and delete data from lists and saved searches of properties, also sorting in the lists by certain parameters.

Problem: Different search bar for different kinds of data, such as people and properties.

Solution: Universal Search bar, allows user to look for any data within the app.
